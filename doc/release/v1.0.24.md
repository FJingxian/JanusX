# JanusX v1.0.24 Release Notes

Compared with v1.0.20, v1.0.24 mainly focuses on stabilizing the Rust-first GWAS/GS workflow, improving packed genotype performance, and making launcher behavior more reliable across macOS, Linux, and Windows.

## Added

- Added the new workflow-based `janusx.assoc` and `janusx.gs` stacks, including structured config/runner/result layers and dedicated packed/stream execution routes for `jx gwas` and `jx gs`.
- Added `postgs` for genomic-selection result summarization and visualization.
- Added new benchmark and validation tools, including `ggval`, `bayesbench`, `garfieldbench`, genotype IO benchmarks, and Windows launcher smoke tests.
- Added more Rust-native kernels and helpers for packed BED processing, Haseman-Elston (HE), PCG, GRM, LD, eigendecomposition, score/beam utilities, and related workflow components.
- Added experimental Metal backend hooks for LM/FaST-LMM nightly paths on macOS.

## Changed

- Refactored `jx gwas` and `jx gs` away from the old monolithic Python entry scripts into workflow-oriented implementations with clearer module boundaries.
- Switched more large-scale genotype handling to mmap-backed packed BED and Rust-first execution paths to reduce Python-side overhead and improve scalability on large cohorts.
- Upgraded rrBLUP/GS internals with HE-based initialization, PCG solving, AdamW support, richer progress reporting, and more explicit thread coordination across BLAS and Rust backends.
- Unified backend selection and fallback logic for OpenBLAS, Accelerate, and plain BLAS on macOS, Linux, and Windows.
- Improved the Rust launcher with safer staged replacement, runtime ownership tracking, and more robust install/upgrade/uninstall behavior on Windows.
- Refreshed project documentation, packaging scripts, and release automation, and aligned the documented Python baseline with Python 3.10+.

## Fixed

- Fixed multiple OpenBLAS detection, linking, and fallback regressions affecting Windows wheels, launcher source upgrades, and mixed-backend builds.
- Fixed a bug that prevented `adamixture` from running correctly when `k=1`.
- Fixed thread-count propagation issues in GS/rrBLUP workflows, including single-core/all-core benchmarking behavior and backend oversubscription edge cases.
- Fixed packed BED regressions in GWAS, GS, and FarmCPU workflows, including metadata mismatch cases, incorrect packed-path triggering, and redundant loading/copy problems.
- Fixed memory and performance issues in rrBLUP PCG flow and packed genotype loading, reducing unnecessary peak RSS during large runs.
- Fixed platform-specific BLAS and launcher issues on macOS and Windows, including slower backend selection, memmap-related build/runtime bugs, and launcher self-update/uninstall edge cases.
- Fixed several downstream workflow bugs in PCA output, LD calculation, GS/postGS cleanup, packaging, and CI validation.

相较于 v1.0.20，v1.0.24 主要聚焦于稳定 Rust-first 的 GWAS/GS 工作流、提升 packed genotype 路径的性能，并增强 launcher 在 macOS、Linux 与 Windows 上的可用性与一致性。

## 新增

- 新增基于 workflow 的 `janusx.assoc` 与 `janusx.gs` 架构，为 `jx gwas` 和 `jx gs` 提供了更清晰的配置、执行、结果分层，以及独立的 packed/stream 执行路径。
- 新增 `postgs` 模块，用于基因组选择结果的汇总与可视化。
- 新增更多 Rust 原生内核与辅助组件，覆盖 packed BED 处理、Haseman-Elston (HE)、PCG、GRM、LD、特征值分解等相关流程。
- 在 macOS 上为 LM/FaST-LMM 的 nightly 路径新增了实验性的 Metal backend 接口。

## 变更

- 将 `jx gwas` 与 `jx gs` 从早期单体式 Python 脚本重构为 workflow 化实现，模块边界更清晰，后续维护和扩展更容易。
- 将更多大规模基因型处理切换到基于 mmap 的 packed BED 与 Rust-first 执行路径，降低 Python 侧开销，提升大样本数据集的可扩展性。
- 升级 rrBLUP/GS 内部实现，引入基于 HE 的初始化、PCG 求解、AdamW 支持、更丰富的进度输出，以及更明确的 BLAS/Rust 线程协同策略。
- 统一了 macOS、Linux 与 Windows 上 OpenBLAS、Accelerate 和 plain BLAS 的后端选择与 fallback 逻辑。
- 强化了 Rust launcher 的 staged replacement、runtime ownership tracking，以及 Windows 下更稳健的安装、升级、卸载行为。
- 更新了项目文档、打包脚本与 release 自动化流程，并将文档中的 Python 版本基线统一到 Python 3.10+。

## 修复

- 修复了多项 OpenBLAS 检测、链接与 fallback 回归问题，影响范围包括 Windows wheel、launcher 源码升级，以及混合后端构建。
- 修复了 `adamixture` 在 `k=1` 时无法正常计算的问题。
- 修复了 GS/rrBLUP 工作流中的线程数传递问题，包括单核/多核 benchmark 行为异常以及后端过度抢占线程的边界情况。
- 修复了 GWAS、GS 与 FarmCPU 中多处 packed BED 路径回归问题，包括 metadata 长度不匹配、错误触发 packed fast path、以及冗余加载/拷贝问题。
- 修复了 rrBLUP 的 PCG 流程与 packed genotype 加载中的内存和性能问题，降低了大规模运行时不必要的峰值 RSS。
- 修复了 macOS 与 Windows 上多项 BLAS 和 launcher 平台问题，包括较慢的后端选择、memmap 相关构建/运行异常，以及 launcher 自升级/卸载边界问题。
- 修复了 PCA 输出、LD 计算、GS/postGS 清理、打包流程与 CI 验证中的若干下游问题。
