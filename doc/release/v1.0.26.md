# JanusX v1.0.26 Release Notes

Compared with v1.0.25, v1.0.26 started from the `postgwas` visualization workflow and then accumulated a second round of GWAS/GS/REML consistency cleanup before release. In addition to merged plotting, panel-ratio layout consistency, cleaner logging, and better PDF/export behavior for downstream figure editing, this release also aligns phenotype-scale PVE reporting, simplifies FarmCPU metadata handling, and removes the old corrected-sparse-PVE presentation from the REML summary path.

## Added

- Added `-manh-merge` and `-qq-merge` for drawing multiple GWAS result files into a single merged Manhattan or QQ figure, with per-series color/marker mapping and threshold-aware highlighting.
- Added `-marker/--marker` for explicit scatter-symbol control in both single-file and merged plotting modes. In merged mode, markers can cycle across input files.
- Added support for running merged plots and per-file plots in the same `jx postgwas` task, so one command can now produce both combined figures and individual figures together.

## Changed

- Removed the old `-merge` switch and consolidated merged plotting behavior under `-manh-merge` and `-qq-merge`.
- Reworked postGWAS panel sizing so `-manh`, `-qq`, merged plots, and combined Manhattan+LD figures all follow the same true panel-ratio layout logic based on the drawable plot panel rather than total figure size.
- Updated merged Manhattan/QQ styling so legends are placed at the right side in a single column, threshold-passing points use per-result palettes, and dense merged layers are rasterized while keeping axes/text/legends export-friendly.
- Updated `.postGWAS.log` output to better align with GWAS/GS logging style, including explicit command recording for reproducibility.
- Centralized figure saving and PDF export handling; JanusX now attempts the Cairo PDF backend when available and otherwise falls back automatically to the default Matplotlib PDF backend.
- Renamed GWAS/GS summary and progress labels from generic `PVE` to `PVE(ph)` / `pheno-scale` whenever the reported quantity is phenotype-scale PVE, reducing confusion with raw variance-component ratios.
- Changed GWAS `-global/--global` row-stat reuse to a single in-memory full-sample pass for the current run, instead of persisting `.scanmeta` files to disk.
- Switched FarmCPU main packed loading and QTN packed preload to the GWAS scan-metadata path, so FarmCPU-oriented packed routes no longer depend on the old `.pmeta.*` metadata cache chain.
- Updated the REML narrow-sense reporting path so dense GRM runs use the dense exact REML null fit directly and report phenotype-scale `h2`; the old `PVEc` column/presentation has been removed from the CLI summary tables.
- Standardized GS/Bayes wording and runtime fields around BLUP phenotype-scale PVE, including auto-`r2` messaging and RRBLUP/GBLUP intermediate state names.

## Fixed

- Fixed several QQ plotting inconsistencies so `-qq` and `-qq-merge` now respect `-ylim`, `-scatter-size`, `-marker`, and `-palette` even when Manhattan plotting is not requested.
- Fixed QQ confidence-band rendering so the trumpet/band remains grey and is computed only once in merged QQ mode.
- Fixed Manhattan and QQ axis-tick behavior so Manhattan y ticks and QQ x/y ticks stay on integer values in both single and merged figures.
- Fixed `-bimrange` display clipping so the shown right boundary no longer extends past the actual chromosome or physical interval limit.
- Fixed combined Manhattan+LD figure layout so the top Manhattan panel now uses the same panel-ratio logic as standalone Manhattan plots.
- Fixed merged/single coexistence and related fallback behavior in `postgwas`, reducing duplicated plotting paths and improving task robustness on restricted multiprocessing environments.
- Fixed a small FvLMM vs dense exact REML / `splmm-exact` PVE discrepancy by making the spectral null-model report consume the same phenotype-scale null `pve` field.
- Fixed sparse/dense GWAS null-fit outputs so they now retain both phenotype-scale PVE (`pve_pheno_scale`) and the raw variance-component ratio (`pve_vc_ratio_raw`) for downstream debugging and consistency checks.
- Fixed FarmCPU/QTN packed genotype preparation so normal GWAS routes no longer generate dominance-layer or packed-metadata sidecar files such as `.pmeta.dom_af.npy`, `.pmeta.row_maf.npy`, `.pmeta.row_missing.npy`, `.pmeta.row_flip.npy`, `.pmeta.site_keep.npy`, `.pmeta.std_denom.npy`, or related `.pmeta.json`.
- Fixed GS result aggregation so RRBLUP/GBLUP exact, lambda-derived, train-variance, and VC-derived PVE values are tracked separately but reported consistently on the phenotype scale.

## Performance

- Slimmed in-memory packed/global GWAS scan metadata by removing the persistent `missing_count` array and letting `af`/`maf` share one `float32` buffer whenever they represent the same per-site quantity.
- Verified on the cubic dataset with plotting disabled (`8,665,855` active SNPs in the global row-stat payload) that FvLMM and SparseLMM did not show a runtime regression, while the resident metadata footprint dropped from about `206.6 MiB` to `140.5 MiB` for that payload, saving roughly `66.1 MiB`.

---

相较于 v1.0.25，v1.0.26 从 `postgwas` 可视化工作流开始，随后又累计了一轮面向正式发布前的 GWAS/GS/REML 一致性清理。除了合并绘图、panel-ratio 布局一致性、更干净的日志输出，以及面向后续图形编辑的 PDF/导出行为改进之外，本版还统一了 phenotype-scale PVE 的命名与尺度、简化了 FarmCPU 的元数据路径，并从 REML 汇总输出中移除了旧的稀疏 PVEc 表达。

## 新增

- 新增 `-manh-merge` 与 `-qq-merge`，可将多个 GWAS 结果文件绘制到同一张 merged Manhattan 或 QQ 图中，并支持按结果文件分配颜色/点形以及阈值以上位点高亮。
- 新增 `-marker/--marker`，用于显式控制单文件模式和 merged 模式下的散点形状；在 merged 模式下可按输入文件循环使用多个 marker。
- 新增在同一次 `jx postgwas` 任务中同时输出 merged 图和逐文件图的能力，因此现在一条命令可以同时产出组合图与单独图。

## 变更

- 移除了旧的 `-merge` 开关，并将合并绘图统一收敛到 `-manh-merge` 与 `-qq-merge`。
- 重构了 postGWAS 的 panel 尺寸逻辑，使 `-manh`、`-qq`、merged 图以及 Manhattan+LD 组合图都基于真正的绘图区 panel，而不是整张 figure 尺寸，来解释 ratio。
- 更新了 merged Manhattan/QQ 的绘图样式：legend 统一放在右侧单列；阈值以上位点按结果文件使用各自配色；高密度 merged 散点层默认栅格化，同时保留坐标轴、文字和 legend 的后期编辑友好性。
- 更新了 `.postGWAS.log` 的输出风格，使其更接近 GWAS/GS 的日志样式，并增加了命令本身的记录，便于复现。
- 统一了图像保存与 PDF 导出逻辑；当环境中可用时，JanusX 会优先尝试 Cairo PDF 后端，否则自动回退到默认的 Matplotlib PDF 后端。
- 将 GWAS/GS 中原本泛化的 `PVE` 标签进一步明确为 `PVE(ph)` / `pheno-scale`，仅在实际输出的是 phenotype-scale PVE 时使用该命名，减少与原始方差组分比值的混淆。
- 将 GWAS 的 `-global/--global` 行统计复用方式改为“本次运行内只做一次全样本 row-stat 计算并在内存中复用”，不再向磁盘持久化 `.scanmeta` 文件。
- 将 FarmCPU 主 packed load 与 QTN packed preload 统一切到 GWAS scan metadata 路径，FarmCPU 相关 packed 路线不再依赖旧的 `.pmeta.*` 元数据缓存链。
- 调整了 REML 窄义遗传力的汇报路径：dense GRM 情况下直接使用 dense exact REML null fit，并以 phenotype-scale `h2` 汇报；CLI 汇总表中移除了旧的 `PVEc` 列/表达。
- 统一了 GS/Bayes 中围绕 BLUP phenotype-scale PVE 的术语与运行态字段，包括 auto-`r2` 提示文案，以及 RRBLUP/GBLUP 的中间状态命名。

## 修复

- 修复了多处 QQ 绘图不一致问题，使 `-qq` 与 `-qq-merge` 即使在未请求 Manhattan 图时，也能正确受 `-ylim`、`-scatter-size`、`-marker` 和 `-palette` 控制。
- 修复了 QQ 置信带的渲染行为，使喇叭口始终保持灰色，并且在 merged QQ 模式下只计算一次。
- 修复了 Manhattan 与 QQ 坐标轴 tick 的问题，使 Manhattan 的 y 轴 tick 以及 QQ 的 x/y 轴 tick 在单图和 merged 图中都尽量落在整数值上。
- 修复了 `-bimrange` 的显示裁剪问题，防止图中右边界超出真实染色体或物理区间上限。
- 修复了 Manhattan+LD 组合图的布局，使顶部 Manhattan panel 现在与单独 Manhattan 图采用相同的 panel-ratio 逻辑。
- 修复了 `postgwas` 中 merged 与 single 模式共存时的分支与 fallback 行为，减少了重复绘图路径，并提升了在受限多进程环境下的任务稳健性。
- 修复了 FvLMM 与 dense exact REML / `splmm-exact` 之间的一点 PVE 细微偏差，使 spectral null-model 的汇报直接复用相同的 phenotype-scale null `pve` 字段。
- 修复了 sparse/dense GWAS null fit 的输出结构，使其同时保留 phenotype-scale PVE（`pve_pheno_scale`）与原始方差组分比值（`pve_vc_ratio_raw`），便于后续调试与尺度一致性核对。
- 修复了 FarmCPU/QTN packed genotype 准备路径，使常规 GWAS 路线不再生成 `.pmeta.dom_af.npy`、`.pmeta.row_maf.npy`、`.pmeta.row_missing.npy`、`.pmeta.row_flip.npy`、`.pmeta.site_keep.npy`、`.pmeta.std_denom.npy` 以及相关 `.pmeta.json` 等 sidecar 文件。
- 修复了 GS 结果聚合逻辑，使 RRBLUP/GBLUP 的 exact、lambda、train-variance 与 VC-derived 多种 PVE 现在会分开保留，但统一按 phenotype-scale 解释与汇报。

## 性能

- 瘦身了 packed/global GWAS 路径中的常驻 scan metadata：移除了持久 `missing_count` 数组，并在 `af` 与 `maf` 实际表示相同位点统计量时共享同一份 `float32` 缓冲区。
- 在 cubic 数据上关闭绘图后的实测中（global row-stat payload 含 `8,665,855` 个 active SNP），FvLMM 与 SparseLMM 未见速度回退；对应 metadata 常驻体积约从 `206.6 MiB` 降到 `140.5 MiB`，节省约 `66.1 MiB`。
